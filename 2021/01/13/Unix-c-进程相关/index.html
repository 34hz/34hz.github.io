进程

创建进程

函数原型

  • 成功调用将会在子进程中返回0,在父进程中返回创建出来的子进程PID
  • 失败返回-1;

    伪代码

函数的调用者可以根据函数返回值的不同,分别为父子进程分配不同的任务,并发执行(多核处理器)

fork() 失败返回-1的条件

1.系统总线程数达到上限

2.系统中总进程数达到上限

子进程与父进程的联系

  • 子进程是父进程的不完全副本,子进程是的数据区BSS区,堆栈区(包括IO流缓冲区,甚至是命令行参数和环境变量都从父进程拷贝,唯有代码区与父进程共享)
  • 当fork()函数成功返回了,父子进程就各自独立运行,其被调用度的先后顺序并不确定,在某些版本的UNIX实现上可以保证子进程先被调度,但是再标准上并没有规定这一点.
  • 当通过fork()创建子进程时,系统内核为父进程维护的文件描述符表,也被复制到子进程的进程表项中,但文件表项并不被复制.例如:在fork()之前父进程如果打开了一个文件,然后创建子进程,这时,在子进程中也可以对该文件进行修改.

vfork()与fork()的区别

  1. vfork()函数创建的子进程不复制父进程的屋里内存,也不拥有自己的内存映射,而是与父进程共享全部的内存空间
  2. vfork()函数会在创建子进程的同时同时挂起父进程,直到进程终止,或通过exec()函数启动了另一个进程,再恢复父进程的运行
  3. 使用vfork()创建的子进程在退出是不要使用return语句,也不要调用exec()函数,而要调用_exec()函数,以避免对父进程造成不利影响
    写时拷贝(copy-on-write)
    现代版本的UNIX系统通常用写时拷贝技术优化性能,依次避免内存复制所带来的系统开销.例如:通过fork()创建进程时,在初始时,并不复制父进程的物理内存,只复制它的内存映射表即可,父子进程共享同一个地址空间,直到父子进程中的一个需要写数据时再复制内存.应用了这种优化机制的fork()函数在性能上并不比vfork()差多少

    使用场景

    fork()
  • 一个进场如果希望创建自己的副本并执行同一份代码,或希望与另一个进程并发的运行,都可以使用fork()函数.
  • 调用fork()前的代码只有父进程执行,fork()函数成功返回后的代码父子进程都会执行,受逻辑控制(根据返回值的判断)会执行不同的分支
    vfork()
    使用vfork()创建的子进程中调用exec()函数启动另一个进程取代自身,比调用fork()要快,因为少了很多东西的复制

进程终止

正常终止(从main函数中返回)

  • 从main函数return可令进程终止,进程是内存中代码和数据的集合,执行代码的过程即线程,每个进程可以有一个到多个线程,单至少要有一个主线程,每个线程都可以被看做是一个独立的执行过程中调用了一个特殊的函数,即线程过程函数.线程开始,线程过程函数被调用,线程过程函数一旦返回,线程即告终止.
    缺省情况下,进程主线程的线程过程函数就是main函数,main函数一旦返回,主线程即终止,主线程一旦终止,进程即终止,进程中如果还有其它线程,其它线程也随之终止.
    main函数的返回值(进程退出码),父进程可以在回收子进程的同时获得其退出码,并根据退出码判断该子进程终止的原因.
  • pthread_exit()退出
    如果在主线程中调用pthread_exit()函数,主线程即终止,进程也就终止了
  • exit()/_exit()/Exit() 的调用
    在程序宏的任何线程中调用了exit() _exit() Exit()都会导致进程终止
进程退出后的资源处理

进程一旦被终止,被终止进程在用户空间所持有的资源会被自动释放,如代码区,数据区,堆栈区等,单内核空间中与该进程有关的资源,如进程表项,文件描述符表等,未必会被立即被释放.

进程的异常终止

  • 在主线程外部,通过pthread_cancel()函数将主线程取消掉,进程即将被终止
  • 当进程执行了某些在操作系统看来具有危险性的操作,或系统本身发生了某种故障或意外,内核向相关进程发送特定的信号.如果进程无意针对所收到的信号采取补救措施,那么内核按缺省的方式将进程杀死,并视情况生成转储文件(core)以备份是否分析,俗称 吐核

    一方面通过回收子进程的僵尸避免资源的过度消耗,另一方面还可以通过子进程僵尸中的某些信息,获得子进程终止的原因.

    相关函数

    等待并回收任意子进程(阻塞)
  • 父进程在创建了若干个子进程以后,调用wait()函数,若所有子进程都在运行,则阻塞,直到所有子进程终止;
  • 若有一个子进程已终止,则返回该子进程的PID并通过status参数(参数非NULL),输出其状态;
  • 若没有要等待的子进程(没有僵尸),函数会返回-1,并置errno为ECHILD
分析进程的终止状态(status参数)

WIFEXITED(status)

  • 若子进程正常终止,值为1;
    WEXITSTATUS(status) status的二进制数位的低8位即退出码
    main函数的返回值和exit()/_exit()/_Exit()函数的参数最好不要超过[-128,127]区间所表示的范围
  • 若子进程异常终止,值为0;
    WTERMSIG(status) status的二进制数位的高24位为导致子进程异常终止的信号
等待并回收任意或特定子进程(可选择是否阻塞)

参数
pid 等待的特定的进程的PID可选以下值

  • pid<-1 等待并回收特定进程组(|pid|)的任意子进程
    回收以该pid绝对值的进程ID为根进程的所有相关进程
  • pid=-1 等待回收任意子进程(等价于wait()函数)
    回收所有调用进程所创建的子进程
  • pid=0 等待并回收与调用进程处在同一进程组的任意子进程
  • pid>0 等待并回收由pid所标识的特定子进程

status 输出子进程的退出状态,可置为NULL
options -可选标志,可取以下值

  • options=0 阻塞模式,若所等子进程仍在运行,则阻塞,直到其终止
  • WNOHANG 非阻塞模式,若所等子进程仍在运行,返回0

创建新进程(exec函数族)

exec与fork区别

exec()与fork()或vfork()不同,exec()函数不是调用进程的子进程,而是创建一个新的进程取代调用进程自身,新进程会用自己的地址空间覆盖调用进程的地址空间,但是进程的PID保持不变

exec创建新进程的变化与延续

调用exec函数不仅改变了调用进程的地址空间和进程映像,调用进程的一些属性也会发生变化,有一些属性也会被继承

变化的属性
  • 任何处于阻塞状态的信号都会丢失
  • 任何被设置为捕获的信号都会还原为默认操作
  • 所有有关线程属性的设置会还原为缺省值
  • 所有当前进程的统计信息全部复位
  • 所有与当前进程内存相关的任何数据都会丢失,包括内存映射文件
  • 所有标准库在用户空间维护的一切数据结构(如通过atexit()/on_exit()注册的退出处理函数)都会丢失
    继承的属性
  • PID PPID 不变
  • 实际用户ID和实际组ID不变
  • 进程优先级不变
  • 默认状态下文件描述符是被继承的(已经打开的文件描述符就不用重新打开了),除非通过fcntl()为文件描述符设置了FD_CLOSEEXEC 的标志位,新进程中会关闭已经打开文件描述符

    exec + vfork模式

    如果既想创建新进程,同时又希望原来的进程继续存在,则可以用vfork()+exec()模式,即在vfork()所创建的子进程中调用exec()函数,新进程取代了子进程,但是其父进程依然存在
    这样就创建了两个代码区完全不同单有父子关系的两个进程

    相关函数

    exec()函数族一共六个函数,他们功能是一样的,用法也接近,只是参数的形式和数量略有不同
    作用都是创建一个新的进程取代调用进程自身,新的进程会用自己的全部地址空间,覆盖调用进程的地址空间,但PID保持不变

返回值
成功不返回;
出错返回-1

execle()

描述

  • 以文件路径找文件
  • 以变长列表给参数
  • 以给定环境变量为PATH

参数
path 可执行文件路径
arg 命令行参数,以NULL结束
envp[] 环境变量数组,以NULL结尾,附带环境变量,以指定的环境变量数组为环境变量,新进程拥有独立的环境变量
